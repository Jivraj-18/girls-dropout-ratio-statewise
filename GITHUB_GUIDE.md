# GitHub Repository Organization Guide

This document specifies what should be committed to Git for a clean, organized repository.

---

## ✅ FILES TO COMMIT (Essential)

### Documentation (must have)
```
README.md                   # Project overview, quick start, commands
PRESENTATION.md             # Main deliverable (ministerial briefing)
DELIVERABLES.md             # Guide to using all outputs
METHODOLOGY.md              # Technical documentation of calculations
```

### Code & Configuration (must have)
```
udise_pipeline.py           # Main analysis script (single entry point)
requirements.txt            # Python dependencies
.gitignore                  # Specifies what not to commit
```

### Supporting Documentation (recommended)
```
docs/
  ├── ANALYSIS_GUIDE.md     # Deep-dive methodology
  ├── DATASET_ASSESSMENT.md # Data quality report
```

### Version Control Files (auto-generated)
```

.gitignore                  # Exclude patterns (already set up)
```

**Total size of committed files:** ~100 KB (very small, efficient for GitHub)

---

## ❌ FILES TO EXCLUDE (Git-Ignored)

### Generated Analysis Outputs
```
outputs/                    # All analysis outputs
├── findings_*.md
├── status_quo_forecast_*.csv
├── udise_state_caste_year_flow_rates.{csv,parquet}
└── charts/
    ├── *.png
    └── *.jpg
```

**Why:** Generated by running the script, can be regenerated anytime. Also, large PNG files increase repo size unnecessarily.

### Raw Data Files
```
udise_api_dumps/            # Downloaded UDISE API responses
├── tabular_map117_stateall_year*.json
├── tabular_map117_national_year*.json
├── master_get_state_year*.json
└── _requests_and_responses.json
```

**Why:** Large files (300+ MB), proprietary data from UDISE+, can be re-downloaded via API.

### Python Environment
```
.venv/                      # Virtual environment
venv/
env/
```

**Why:** Large directory (~500 MB), environment-specific, recreatable via `pip install -r requirements.txt`.

### Compiled Python & Cache
```
__pycache__/                # Python bytecode
*.pyc
*.pyo
*.egg-info/
```

**Why:** Generated during execution, can be regenerated.

### IDE & OS Files
```
.vscode/                    # VS Code settings (personal)
.idea/                      # PyCharm settings (personal)
*.swp, *.swo                # Editor temp files
*~                          # Backup files
.DS_Store                   # macOS metadata
Thumbs.db                   # Windows thumbnail cache
```

**Why:** Personal/environment-specific, cloud services (GitHub Codespaces) handle differently.

### Temporary Files
```
*.log                       # Log files
*.tmp                       # Temporary files
.cache/                     # Cache directories
```

**Why:** Temporary, not needed for reproduction.

## 🚀 GitHub Workflow

### To Set Up the Repository

1. **Initialize:**
   ```bash
   cd /home/jivraj/straive/girl-dropout-rate-analysis
   git init
   ```

2. **Add files:**
   ```bash
   git add README.md PRESENTATION.md DELIVERABLES.md METHODOLOGY.md udise_pipeline.py requirements.txt .gitignore docs/
   ```

3. **First commit:**
   ```bash
   git commit -m "Initial commit: Girls' education dropout analysis"
   ```

4. **Add remote:**
   ```bash
   git remote add origin https://github.com/[username]/girl-dropout-rate-analysis.git
   git branch -M main
   git push -u origin main
   ```

### To Deploy to GitHub

```bash
# Daily development workflow
git add <changed-files>
git commit -m "Descriptive message"
git push

# Commit messages should follow:
# - "docs: Update presentation findings"
# - "feat: Add new analysis section"
# - "fix: Correct calculation method"
# - "chore: Update with new UDISE data (years 16-23)"
```

### To Collaborate

1. **Contributor clones:**
   ```bash
   git clone https://github.com/[username]/girl-dropout-rate-analysis.git
   cd girl-dropout-rate-analysis
   ```

2. **Setup environment:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   python -m playwright install chromium
   ```

3. **Generate outputs locally:**
   ```bash
   python udise_pipeline.py all  # Creates outputs/ (won't be committed)
   ```

4. **Make changes to documentation or code:**
   ```bash
   # Edit PRESENTATION.md or udise_pipeline.py
   git add <changed-files>
   git commit -m "Improve analysis documentation"
   git push
   ```
